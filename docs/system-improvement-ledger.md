# System Improvement Ledger — beacon-design

Spoke friction that turned out to be the system's fault, one line each: what was
missing, where it bit, the proposed hub fix. Reviewed for promotion to the hub.

- 2026-07-15 — **manifest cross-check can't see PascalCase spoke components**: `scripts/lib/manifest-crosscheck.mjs` `importToKebab()` lowercases the import basename (`BcnProjectHeader` → `bcnprojectheader`), which can never match a kebab manifest resolver (`bcn-project-header`), so every manifest'd page in this spoke throws false `manifest-declared-absent` errors (bit on `project-dashboard.astro` — 6 false errors in /design-qa). Proposed hub fix: `return pascalToKebab(base)` at `manifest-crosscheck.mjs:52` (a no-op for the hub's already-kebab basenames).
- 2026-07-15 — **check-adherence token scan reads `var(--…)` inside comments**: PageLayout.astro's frontmatter doc comment (`$gray-N → var(--bcn-gray-N)`) produced two false `token-undefined` errors. Worked around by rewording the comment. Proposed hub fix: strip comments before the token scan.
- 2026-07-15 — **composition-heft heuristic fires on section components**: the >120-style-line "page is hand-rolling" warning (documented as per-authored-PAGE) flagged `BcnStarredComponents` / `BcnTrackingZone`, which are section components that legitimately own their CSS. Proposed hub fix: apply the heuristic only to files under `src/pages/`.
- 2026-07-15 — **no neutral count-badge variant in `esa-badge`**: six saturated-fill variants, no border option, green `primary` as default — Andy wants a neutral gray-border count as the system DEFAULT and the green excised. Bit on project-dashboard (composed in-spoke as `BcnCount` meanwhile). Tracked as margot thread `thr-mrmjb7v7-1f211c7b`. Proposed hub fix: add the neutral variant, make it the count default.
- 2026-07-15 — **RESOLVED hub-side**: the manifest cross-check PascalCase gap above is fixed in ecology `c47d3de` (`importToKebab` now kebabs basenames), together with a second false positive found the same way: web-component side-effect imports in body `<script>` blocks (the only place they can live — frontmatter would execute the element during SSR) now count as composition imports. Both pinned with tests. Bit again on `fish-gantt.astro` + reproduced on shipped `permit-tracking.astro` (6 false errors) before fixing.
- 2026-07-15 — **banned-border-left rule fires on chart lines**: `check-adherence` flags Gantt gridlines, milestone dashes, and the today-line — zero-width `border-left` elements inside the plot — as "colored left-border status indicators," which they aren't (the ban targets status/category edges on cards/rows/containers). Bit on `fish-gantt` (`BcnWorkBreakdown`, `BcnRollupSummary` — 5 false errors). Worked around by drawing the lines as `width:1-2px; background` / `box-shadow` / `repeating-linear-gradient`. Proposed hub fix: exempt `width: 0`/line-shaped elements, or scope the rule to selectors that also carry card/row/container classes.
- 2026-07-15 — **decomposition-context.mjs returns an empty `spokeInventory` in a git worktree**: running the grounding script from the `beacon-design--ws-fish` worktree produced 0 spoke components, so `reuseHints` was blank and the decomposition reviewer had to do its reuse comparison by reading files directly. Proposed hub fix: make the inventory scan worktree-safe (likely resolving paths through `.git` file indirection or the repo scan root).
- 2026-07-15 — **no size variant on `esa-stat`**: the Study Planning KPI strip needed a compact stat; page-side `:global` overrides shrink value/label/sub. Proposed hub fix: `size="sm"` prop on esa-stat.
- 2026-07-15 — **the served token sheet dropped the legacy named ramps**: `@esa/tokens`'s package export maps `tokens.css` → `dist/tokens.css`, which ships Radix-style ramps (`--color-green-1..12`) but NOT the legacy names (`--color-green-700`, `--color-orange-500`, `--color-blue-500/700`) that `theme-beacon.css`'s re-points were written against — so `--color-success`, `--color-warning`, `--color-ai`, `--color-ai-hover` resolved to NOTHING at runtime (on-track/attention pulse dots rendered transparent; Andy caught it as "why do some TMR have dots and others don't"). Worked around by inlining the four values in the theme. Proposed hub fix: either ship the legacy named ramps in `dist/tokens.css` or add a build-time check that every spoke-theme `var()` target exists in the served sheet (check-adherence's vocab scan reads the whole package tree, so it missed this — it should read ONLY the sheets the export map actually serves).
