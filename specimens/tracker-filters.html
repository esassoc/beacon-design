Requirement Tracker — Filter Area

Three directions for adapting the tracker's filter zone to match Beacon prod: a top bar (View tabs · N/A · keyword Search) over a row of multi-select filters — Type, Phase, Commitment, List, Species, Construction Activities, Milestones. Scopes removed; the small type-pills removed; Type & Phase added. This intentionally drops our standard "search box + Clear Filters" pattern in favor of dropdown facets.

Specimen · standalone HTML · not wired to the grid. Click any filter to see its checkbox panel. Pick a direction and I'll build it from esa-* legos.

What to react to — the three differ mainly in how heavily the filter zone is framed and how active selections read:
  • A — Carded two-band: closest to prod. Bordered card, labeled groups w/ dividers, "Filters" row underneath. Familiar, but tallest (eats grid height).
  • B — Flat toolbar: no card chrome. View+search+N/A on one line, dropdowns on the next. Lightest, gives the most vertical room to the grid (ties into ask #2).
  • C — Tinted filter strip: middle ground. Search-led top row, then filters grouped by a soft teal background instead of a border — reads as a distinct "filter zone" without a heavy box.
Option A

Carded two-band — prod parity

Mirrors the current Beacon screen most literally. Top band groups View / N/A / Search with vertical dividers; the filter band sits below a divider with a leading Filters label. Active facets show an inline count badge. Trade-off: most chrome, tallest footprint.

Option B

Flat toolbar — max grid room

Strips the card. View toggle + N/A on the left, search pushed right; the filter dropdowns flow on a second line behind a funnel glyph — no "Filters" label, no box. Lowest height, so the AG Grid gets the extra rows from ask #2. Active selections collapse into removable pills only when present.

Option C

Tinted filter strip — grouped without a box

Search-led: View + N/A on the left, a wide search filling the top row. The seven filters live inside a soft teal strip — grouped by background, not a border — so the "filter zone" reads as one unit while staying light. A Clear all link appears at the strip's end when anything is active.

Lego mapping (all three build from the same hub components)
  • View tabs → esa-button-toggle · N/A → esa-switch-toggle · Search → esa-text-field
  • Each filter → esa-filter-dropdown (multi-select + checkbox panel — the Beacon ui-filter equivalent), wrapped in esa-filter-container
  • Active pills → esa-filter-pills · Clear all → esa-filter-clear-button
  • Filter order is fixed: Type · Phase · Commitment · List · Species · Construction Activities · Milestones
  • Ask #2: grid height grows by ~10 rows (calc(100vh − 420px) → smaller offset / more rows). Option B's shorter filter zone buys the most room for free.
